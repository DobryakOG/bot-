# Discord Bot - Система Заявок

Бот для автоматизации приема заявок на вступление в семью на Discord сервере.

## Возможности

- 📝 Автоматическое создание тикетов для каждой заявки
- 👀 Система рассмотрения заявок с кнопками управления
- 🔊 Вызов на собеседование в голосовой канал
- 📊 Логирование всех заявок с цветовой индикацией статуса
- 💬 Автоматические уведомления в ЛС пользователям
- 🎨 Поддержка изображений (баннер)

## Установка

1. Установите Python 3.8 или выше

2. Установите необходимые библиотеки:
```bash
pip install discord.py python-dotenv
```

3. Создайте бота на [Discord Developer Portal](https://discord.com/developers/applications)

4. Включите следующие Privileged Gateway Intents:
   - SERVER MEMBERS INTENT
   - MESSAGE CONTENT INTENT

5. Пригласите бота на сервер с правами:
   - Manage Channels
   - Send Messages
   - Embed Links
   - Attach Files
   - Read Message History
   - Manage Messages
   - Create Private Threads

## Настройка

1. Откройте файл `.env` и заполните все необходимые данные:

```env
DISCORD_TOKEN=ваш_токен_бота
APPLICATION_CHANNEL_ID=id_канала_для_объявления
TICKET_CATEGORY_ID=id_категории_для_тикетов
LOG_CHANNEL_ID=id_канала_для_логов
REVIEWER_ROLE_IDS=id_роли1,id_роли2
INTERVIEW_VOICE_CHANNELS=id_голосового_канала1,id_голосового_канала2
```

2. (Опционально) Добавьте изображение баннера:
   - Поместите файл `banner.png` в папку `images/`
   - Поддерживаются форматы: PNG, JPEG, GIF

## Как получить ID

1. Включите режим разработчика в Discord:
   - Настройки → Расширенные → Режим разработчика

2. Щелкните правой кнопкой мыши на канал/роль/категорию → Копировать ID

## Запуск

```bash
python bot.py
```

## Первоначальная настройка на сервере

После запуска бота выполните команду на сервере:
```
/setup_applications
```

Эта команда разместит сообщение с кнопкой "Заполнить анкету" в указанном канале.

## Как работает система

### Для пользователей:
1. Нажимают кнопку "Заполнить анкету"
2. Заполняют форму с вопросами
3. Автоматически создается приватный тикет
4. Получают уведомления о статусе заявки

### Для рекрутеров:
1. **Взять на рассмотрение** - уведомляет пользователя, что заявка в работе
2. **Вызвать на обзвон** - выбирает голосовой канал для собеседования
3. **Принять** - принимает заявку и закрывает тикет
4. **Отклонить** - запрашивает причину отказа и уведомляет пользователя

### Логирование:
- 🔵 Синий - заявка на рассмотрении (никто не взял)
- 🟠 Оранжевый - заявка взята на рассмотрение
- 🟢 Зеленый - заявка принята
- 🔴 Красный - заявка отклонена
